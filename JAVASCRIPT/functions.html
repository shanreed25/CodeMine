Default Parameters

Default parameters, introduced in ECMAScript 2015 (ES6), allow function parameters to be initialized with default values if no value or undefined is passed during a function call.

Provides a concise way to handle missing arguments and improve code readability by eliminating the need for manual checks for undefined values within the function body.

Default parameters are declared by assigning a default value to a parameter in the function signature using the assignment operator (=)

How it works
  • Syntax: function functionName(parameter1 = defaultValue1, parameter2 = defaultValue2, ...) {// Function body}
  • If a function is called and an argument corresponding to a parameter with a default value is omitted, or if undefined is explicitly passed for that argument, the default value will be used for that parameter.
  • If a value other than undefined (e.g., null, a number, a string, an object) is passed for a parameter with a default value, the passed value will override the defaul

Example

  function greet(name = 'Guest', message = 'Hello')  {
     console.log(`${message}, ${name}!` );
 }

greet(); // Output: Hello, Guest!
greet('Alice'); // Output: Hello, Alice!
greet('Bob', 'Hi'); // Output: Hi, Bob!
greet(undefined, 'Greetings'); // Output: Greetings, Guest!
greet(null); // Output: Hello, null! (null is a valid value, not undefined)
Benefits
  • Readability and Conciseness: Reduces boilerplate code for handling missing arguments.
  • Error Prevention: Helps prevent errors that can arise from operating on undefined values.
  • Provides Default Behavior: Ensures functions have predictable behavior even when not all arguments are provided.
