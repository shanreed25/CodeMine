Events

Events are actions or occurrences that happen in the browser. They are triggered by user interactions or by the browser itself.

The system tells you about these events and you can create JavaScript code that will react to them

Types of Events

  • Mouse Events: Triggered by mouse interactions, such as click, onclick, mousedown, mouseup, mousemove, mouseenter, and mouseleave.
  • Keyboard Events: Triggered by keyboard interactions, such as keydown, keyup, and keypress.
  • Form Events: Triggered by form interactions, such as input, change, submit, focus, and blur.
  • Window Events: Triggered by window actions, such as load, resize, scroll, and beforeunload.
  • Document Events: Triggered by document changes, such as DOMContentLoaded and visibilitychange.
  • Clipboard Events: Triggered by clipboard actions, such as copy, paste, and cut.
  • Drag and Drop Events: Triggered by drag and drop actions.
  • Media Events: Triggered by media actions.
  • Focus and Blur Events: Triggered when an element gains or loses focus

When the system tells us about the events, how do we get access to this "message".

Javascript Event Handling Methods.

Event Handling Methods

Event handlers allow developers to define how a web page should respond to events.

When an event occurs on a target element, a handler function is executed in response to that event.

How to use event handler functions with events

  1. Element Selection: Select the HTML element to which you want to attach an event handler.
  2. Event Type: Specify the type of event you want to listen for.
  3. Event Handler Function: Define a JavaScript function that will be executed when the specified event occurs on the selected element
  4. Event Binding: Bind the event handler function to the selected element and event type

There are three main ways to bind/add event handlers to HTML elements:

  • Inline Event Handlers:Defined directly in HTML attributes
  • Event Listener Method:Uses builted in Javascript methods to attach and detach event handlers.
  • DOM Property Event Handlers:Assigns event handlers as properties in JavaScript

Event handler functions can also access an event object, which provides information about the event that occurred, such as the target element, the mouse coordinates, or the key that was pressed. This information can be used to customize the behavior of the event handler function.

Event Object

The event object in JavaScript is a built-in object that is automatically created and passed to event handlers when an event occurs.

It contains information about the event, such as the type of event, the target element, and any additional data associated with the event, that can be used within event handlers to respond appropriately.

How it Works

  • When an event occurs, the browser creates an event object
  • The event object is then passed as an argument to the event handler function.
  • The event handler function can access the event object's properties and methods to handle the event.

Example

document.getElementById('myInput').addEventListener('input', function(event) {
      console.log('Event type:', event.type);
      console.log('Target element:', event.target);
      console.log(event)
      event.preventDefault(); // Prevent the default behavior of the event
});

Event objects are essential for handling user interactions and other dynamic behaviors in JavaScript. They provide the necessary context and information to create responsive and interactive web applications. The event object is a fundamental part of event handling in JavaScript, providing a way to access information about events and control their behavior.

Important Event Object Methods

  • preventDefault(): stop default browser behavior
  • stopPropagation(): to halt event propagation

DOM Property Event Handlers

DOM property event handlers are a simple way to respond to events by assigning a function to the element's event property (e.g., onclick, onmouseover).

Sets the element's event property directly to a function

  • Assigns event handlers as properties
  • The property name for event handlers typically starts with "on" followed by the event name, such as "onclick", "onload", or "onfocus"
  • It separates HTML and JavaScript, improving readability
  • Only one handler can be assigned to an event property for a given element

How to use DOM Property Event Handlers

  1. Select the HTML element you want to add the event handler to
    • You can do this using methods like "document.getElementById()" or "document.querySelector()"
    • const myButton = document.getElementById('myButton');
  2. Assign/Attach a function to the corresponding DOM property event handler of the selected element
    • This function will be executed when the specified event occurs
    • myButton.onclick = function() { alert('Button was clicked!');};
    • You can also do this by assigning a pre-defined function
  3. The event object: When the event handler is triggered, it receives an event object as an argument. This object contains useful information about the event, such as the event type, the target element, and coordinates for mouse events.

Key features and considerations

  • This method is straightforward for attaching a single event handler to an element
  • Only one event handler can be assigned per event type to a single element using this method. If you assign a new function to the property, the previous handler will be replaced
  • While still functional, addEventListener() is generally the preferred method for modern web development due to its flexibility and ability to attach multiple event listeners.

Inline Event Handlers

Inline event handlers are defined directly in the HTML element's opening tag of the target element

The event handler function is added as the value to an event handler attribute that start with "on".

How to use inline event handlers

  1. Identify the HTML element
  2. Add a event attribute to the element
  3. As the value of the event attribute, include the JavaScript code you want to execute when the event occurs
    • This code can be a single statement, or you can call a JavaScript function

Example: Calling a JavaScript function

<button onclick= "myFunction()" > Click Me </button> <script>
   function myFunction()  {
     console.log("Button clicked!" );
 }
</script>
  • onclick: is the event handler attribute
  • myFunction(): is the event handler function
  • When the button is clicked myFunction() will be executed

Example: Simple inline JavaScript

<button onclick= "alert('HI!' )" > Click Me </button>
  • onclick: is the event handler attribute
  • alert('HI!'): is the event handler function
  • When the button is clicked an alert box with the message "HI!" will show

Drawbacks of Inline Event Handlers

  • Mixing concerns: They mix HTML and JavaScript, making code harder to maintain and understand
  • Limited flexibility: Only one handler can be attached per event per element, and they cannot be dynamically added or removed
  • Potential security risks: Inline scripts can increase vulnerability to cross-site scripting (XSS) attacks
  • Global scope issues: Functions called from inline handlers must be globally accessible, which can lead to namespace conflicts
  • Accessibility issues: Certain device-dependent inline handlers like onmouseover can cause accessibility problems.

Inline event handlers should be used for simple basic uses but are not recommended for complex applications due to these drawbacks

Alternative approaches:

  • addEventListener(): They mix HTML and JavaScript, making code harder to maintain and understand
    • The standard and recommended way to handle events
    • Allows attaching multiple event listeners to an element without overwriting existing ones
    • Offers more control over event propagation (bubbling/capturing)
  • Frameworks like React and Vue.js: Only one handler can be attached per event per element, and they cannot be dynamically added or removed
    • Component-based approaches manage event handling in a more structured and maintainable way.

Event Listener Method

The addEventListener() method in JavaScript is used to attach an event handler to a specified element. This enables you to execute a function when a particular event occurs on that element

Takes the event name and the function to be executed as arguments. This method is recommended for its power and scalability, allowing multiple handlers, providing more control over event propagation, and being easier to remove with removeEventListener().

Syntax: element.addEventListener(event, function, useCapture );

  • element: The HTML element to which you are attaching the event listener.
  • event: A string specifying the name of the event (e.g., "click", "mouseover", "keydown").
  • function: The function to be executed when the event occurs.
  • useCapture: An optional boolean value specifying whether to use event capturing or bubbling (default is false for bubbling).

Example

<button id = "myButton" > Click Me </button>
<script>
     const button = document.getElementById( 'myButton' );

    button .addEventListener( 'click' , function() {
        alert( 'Button Clicked!' );
    } );
</script>

When the button with the ID myButton is clicked, the alert message "Button Clicked!" will be displayed

Key Features and Benefits

  • Multiple Event Listeners: You can add multiple event listeners to the same element for the same event.
  • Dynamic Handling: You can dynamically add and remove event listeners using removeEventListener()
  • Flexibility: Supports a wide range of events, allowing you to respond to various user interactions and browser events
  • Event Propagation: Allow you to control the order in which event handlers are executed during the capturing and bubbling phases

Important Notes

  • Do not use the "on" prefix when specifying the event name. (e.g., use "click" instead of "onclick")
  • The function passed as an argument to addEventListener can be an anonymous function or a named function
  • The "this" keyword inside the event handler function refers to the element on which the event occurred.
  • The event object is automatically passed as the first argument to the event handler function. It contains information about the event.
