Events

Events are actions or occurrences that happen in the browser. They are triggered by user interactions or by the browser itself.

The system tells you about these events and you can create JavaScript code that will react to them

Types of Events

  • Mouse Events: Triggered by mouse interactions, such as click, onclick, mousedown, mouseup, mousemove, mouseenter, and mouseleave.
  • Keyboard Events: Triggered by keyboard interactions, such as keydown, keyup, and keypress.
  • Form Events: Triggered by form interactions, such as input, change, submit, focus, and blur.
  • Window Events: Triggered by window actions, such as load, resize, scroll, and beforeunload.
  • Document Events: Triggered by document changes, such as DOMContentLoaded and visibilitychange.
  • Clipboard Events: Triggered by clipboard actions, such as copy, paste, and cut.
  • Drag and Drop Events: Triggered by drag and drop actions.
  • Media Events: Triggered by media actions.
  • Focus and Blur Events: Triggered when an element gains or loses focus

When the system tells us about the events, how do we get access to this "message".

Event Object.

Event Object

The event object in JavaScript is a built-in object that is automatically created and passed to event handlers when an event occurs.

When an event occurs on a target element, a handler function is executed in response to that event.

When the event handler is invoked, the event object is passed as an argument to that function. This allows the event handler to access and utilize information about the event.

How it Works

  • When an event occurs, the browser creates an event object
  • The event object is then passed as an argument to the event handler function.
  • Event handler functions are typically registered using "addEventListener()" or assigned to event handler properties like "onclick"
  • The event handler function can access the event object's properties and methods to handle the event.

Event Object Properties and Methods

The event object contains various properties and methods that provide details about the event and allow developers to control its behavior.

Key properties

  • target: the element where the event originated
  • type: the specific type of event
  • timeStamp: Gives the time the event was created in milliseconds
  • currentTarget: the element whose event listener was triggered
  • bubbles: A boolean indicating whether the event will bubble up through the DOM
  • defaultPrevented: shows if preventDefault() has been called
  • data: Contains the optional data passed to an event method when the current executing handler is bound
  • isTrusted: A boolean that indicates whether the event was triggered by the browser (true) or by JavaScript (false)
  • composed: indicates if the event can bubble across the shadow DOM
  • eventPhase: shows the current phase of the event flow

Key methods

  • preventDefault(): stops default action
  • stopPropagation(): stops bubbling
  • stopImmediatePropagation(): prevents other listeners on the same element
  • composedPath(): returns the event's path

Event Handling Methods

Event handlers allow developers to define how a web page should respond to events.

When an event occurs on a target element, a handler function is executed in response to that event.

Event handler functions can access the event object.

There are three main ways to bind/add event handlers to HTML elements:

  • Inline Event Handlers Defined directly in HTML attributes
  • Event Listener Method Uses builted in Javascript methods to attach and detach event handlers.
  • DOM Property Event Handlers Assigns event handlers as properties in JavaScript

Inline Event Handlers

JavaScript inline event handlers involve directly embedding JavaScript code or function calls within HTML attributes to respond to specific events.

This method directly links the presentation layer (HTML) with the behavior layer (JavaScript).

Inline event handlers are defined directly in the HTML element's opening tag of the target element

The event handler function is added as the value to an event handler attribute that start with "on".

How to use inline event handlers

  1. Identify the HTML element
  2. Add a event attribute to the element
  3. As the value of the event attribute, include the JavaScript code you want to execute when the event occurs
    • This code can be a single statement, or you can call a JavaScript function

  • alert('HI!'): is the event handler function

Drawbacks of Inline Event Handlers

  • Mixing concerns: They mix HTML and JavaScript, making code harder to maintain and understand
  • Limited flexibility: Only one handler can be attached per event per element, and they cannot be dynamically added or removed
  • Potential security risks: Inline scripts can increase vulnerability to cross-site scripting (XSS) attacks
  • Global scope issues: Functions called from inline handlers must be globally accessible, which can lead to namespace conflicts
  • Accessibility issues: Certain device-dependent inline handlers like onmouseover can cause accessibility problems.

Inline event handlers should be used for simple basic uses but are not recommended for complex applications due to these drawbacks

Alternative approaches:

  • addEventListener(): They mix HTML and JavaScript, making code harder to maintain and understand
  • Frameworks like React and Vue.js: Only one handler can be attached per event per element, and they cannot be dynamically added or removed
    • Component-based approaches manage event handling in a more structured and maintainable way.

Event Listener Method

The addEventListener() method in JavaScript is used to attach an event handler to a specified element. This enables you to execute a function when a particular event occurs on that element

  • The standard and recommended way to handle events
  • Allows attaching multiple event listeners to an element without overwriting existing ones
  • Offers more control over event propagation (bubbling/capturing)

Takes the event name and the function to be executed as arguments.

Syntax: element.addEventListener(event, function, useCapture );

  • element: The HTML element to which you are attaching the event listener.
  • event: A string specifying the name of the event (e.g., "click", "mouseover", "keydown").
  • function: The function to be executed when the event occurs.
  • useCapture: An optional boolean value specifying whether to use event capturing or bubbling (default is false for bubbling).

How to use event handler functions with events

  1. Element Selection: Select the HTML element to which you want to attach an event handler.
  2. Event Type: Specify the type of event you want to listen for.
  3. Event Handler Function: Define a JavaScript function that will be executed when the specified event occurs on the selected element
  4. Event Binding: Bind the event handler function to the selected element and event type

Key Features and Benefits

  • Multiple Event Listeners: You can add multiple event listeners to the same element for the same event.
  • Dynamic Handling: You can dynamically add and remove event listeners using removeEventListener()
  • Flexibility: Supports a wide range of events, allowing you to respond to various user interactions and browser events
  • Event Propagation: Allow you to control the order in which event handlers are executed during the capturing and bubbling phases

Important Notes

  • Do not use the "on" prefix when specifying the event name. (e.g., use "click" instead of "onclick")
  • The function passed as an argument to addEventListener can be an anonymous function or a named function
  • The "this" keyword inside the event handler function refers to the element on which the event occurred.
  • The event object is automatically passed as the first argument to the event handler function. It contains information about the event.

DOM Property Event Handlers

DOM property event handlers are a simple way to respond to events by assigning a function to the element's event property (e.g., onclick, onmouseover).

How to use DOM Property Event Handlers

preventDefault()

The event.preventDefault() method is a crucial part of handling events with JavaScript

It allows you to stop the browser from performing the default action that would normally occur when a particular event happens.

What it does

  • Cancels the default action: When an event occurs on an element, the browser usually has a built-in behavior associated with that event. For example, clicking a link ( <a>) normally navigates the user to the specified URL. Submitting a form typically refreshes the page.
  • Allows you to override the default behavior: By calling event.preventDefault() within your event handler function, you instruct the browser not to execute that default action. This gives you control to handle the event in a custom way using your JavaScript code.

When to use

  • Preventing form submission: When you want to handle form data with JavaScript (e.g., using AJAX) instead of allowing the default browser form submission.
  • Preventing link navigation: When you want a link to trigger a JavaScript function or action instead of directing the user to a new page.
  • Controlling context menus: You can prevent the browser's default context menu (right-click menu) from appearing and display a custom one instead.
  • Customizing key actions: For example, preventing the default action of certain key presses in input fields to implement custom input validation or behavior.

Important Notes

  • Does not stop event propagation, so the event will still propagate through the DOM tree (unless you also call stopPropagation())
  • Not all events are cancelable, so you check the cancelable property of the event object to see if preventDefault() will have an effect.
  • Consider alternative solutions: MDN suggests looking for better ways to block default actions, such as using disabled or readonly attributes on form controls or HTML constraint validation, rather than relying solely on preventDefault()

# Event bubbling is a fundamental concept in JavaScript that describes the order in which event handlers are executed when an event occurs on an element with nested elements. It is a type of event propagation where an event first triggers on the innermost target element, and then successively triggers on the ancestors (parents) of the target element in the same nesting hierarchy until it reaches the outermost DOM element or document object. When an event occurs on a DOM element, it first goes through the capture phase, where the event travels down the DOM tree from the window to the target element. Then, the event triggers on the target element itself. Finally, the event goes through the bubbling phase, where the event travels back up the DOM tree from the target element to the window. Event bubbling can be useful for a variety of purposes, such as: Centralized event handling: You can attach event listeners to parent elements and handle events that occur on their children. This can simplify your code and make it easier to manage events. Event delegation: You can use event bubbling to implement event delegation, which allows you to handle events on a large number of elements without attaching event listeners to each element individually. Error handling: If an error occurs during the handling of an event at a lower level in the hierarchy, the event will continue to bubble up. This allows you to have centralized error handling at a higher level. However, event bubbling can also cause unexpected problems if you're not careful. For example, if you have two event listeners on different elements that are both triggered by the same event, the event listener on the parent element will be triggered after the event listener on the child element. To prevent event bubbling, you can use the stopPropagation() method of the event object. This method will prevent the event from bubbling up to its parent elements # Event propagation in JavaScript describes the order in which event handlers are executed when an event occurs on an element that has nested elements. There are three phases: capturing, target, and bubbling. Capturing Phase: The event travels down the DOM tree from the window to the target element. Event listeners in capture mode are triggered. Target Phase: The event reaches the target element, and any event handlers attached to the target element are triggered. Bubbling Phase: The event travels back up the DOM tree from the target element to the window. Event listeners in bubble mode are triggered. By default, event listeners are in bubble mode, meaning that the event will bubble up the DOM tree after it reaches the target element. However, you can set an event listener to capture mode by passing true as the third argument to the addEventListener() method: JavaScript element.addEventListener('click', function(event) { // This code will be executed during the capturing phase }, true); You can also use event.stopPropagation() to stop the event from propagating further up the DOM tree during the bubbling or capturing phase. Additionally, event.preventDefault() prevents the default action associated with the event, such as submitting a form or following a link.
