Data Structures

Python provides several built-in data structures that allow for efficient organization and manipulation of data

  • Lists: Ordered, mutable collections of items. They can store heterogeneous data types and are defined using square brackets []
  • Dictionaries: Unordered collections of key-value pairs. Each key must be unique and immutable, while values can be of any type. They are defined using curly braces {}
  • Tuples: Ordered, immutable collections of items. Similar to lists but cannot be modified after creation. They are defined using parentheses ()
  • Sets: Unordered collections of unique items. They do not allow duplicate elements and are useful for operations like union, intersection, and difference. They are defined using curly braces {} or the set() constructor

Python also supports the implementation of user-defined data structures like

  • Stacks: Follow a Last-In, First-Out (LIFO) principle, where the last element added is the first one removed. Can be implemented using lists
  • Queues: Follow a First-In, First-Out (FIFO) principle, where the first element added is the first one removed. Can be implemented using lists or the collections.deque module.
  • Trees: Hierarchical data structures where each node can have multiple child nodes. Binary trees and binary search trees are common examples.
  • Linked Lists: Consist of nodes where each node contains data and a reference to the next node.
  • Graphs: Represent networks of interconnected nodes (vertices) and edges.
