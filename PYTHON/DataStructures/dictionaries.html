Dictionaries

Dictionaries allow us to group together and tag related pieces of information, and functions similarly to a dictionary in real life.

It's a data structure that allows us to associate a key to a value and pair the two pieces of data together

  • Unordered: do not maintain a specific order of elements so, the order in which items are inserted or retrieved is not guaranteed.
  • Mutable: can be modified after creation so, you can add, remove, or change key-value pairs.
  • Key value pairs: each item consists of a unique key mapped to a corresponding value
  • Key Indexed: indexed by keys, not by numerical indices

Every dictionary has two parts

  • Key: equivalent of a word, in a real life dictionary, and must be immutable types (e.g., strings, numbers, tuples)
  • Value: equivalent of the definition of a word, in a real life dictionary, and can be of any data type

Efficient Lookup: Dictionaries are highly optimized for retrieving values based on their keys, offering fast lookup times due to their internal implementation using hash tables.

Syntax

{key: value, key: value, key: value}
